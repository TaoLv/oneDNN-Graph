Build from Source

Check out the Source Code

Check out the branch from oneDNN repository and update the third party modules.

git clone https://github.com/oneapi-src/oneDNN.git --branch dev-graph --recursive

Build the Library

Ensure that all software dependencies are in place and have at least the minimal supported version.

The oneDNN Graph build system is based on CMake. Use

  • CMAKE_BUILD_TYPE to select between build type (Release, Debug, RelWithDebInfo).

Linux/macOS

Prepare the Build Space

mkdir -p build && cd build

Generate makefile

  • Native compilation:
cmake ..
  • Compilation with enabled unittests and examples
cmake .. -DDNNL_GRAPH_BUILD_EXAMPLES=True -DDNNL_GRAPH_BUILD_TESTS=True

Build and Install the Library

  • Build the library:
make -j
  • Build the documentation:
make doc

Windows

  • Generate a Microsoft Visual Studio solution
mkdir build && cd build
cmake .. -G "Visual Studio 16 2019"

To build with unittests and examples

mkdir build && cd build
cmake .. -G "Visual Studio 16 2019" -DDNNL_GRAPH_BUILD_TESTS=1 -DDNNL_GRAPH_BUILD_EXAMPLES=1 -DCTESTCONFIG_PATH=\\PATH\TO\oneDNNGRAPH\build\src\Release
  • Build the Library
cmake --build . --config Release

Validate the Build

If the library is built for the host system, you can run unit tests and examples using:

cd build
ctest -V

Install the Library

To install the built library, you need to have the write privilege of the target directory with sudo or specifying the target directory via -DCMAKE_INSTALL_PREFIX in the cmake command line.

cd build
make install
